# Backend - Centrifungal API & CMS

Laravel 13 + Statamic 6 application serving the product API, cart, checkout, and CMS control panel.

## Prerequisites

- PHP 8.5 (with extensions: pdo_sqlite, gd, bcmath, mbstring, xml, curl, zip, intl)
- Composer 2
- SQLite 3.52.0+
- Node.js (for Vite asset compilation during development)

## Setup

```bash
cd backend
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate
```

To seed the database with sample data:

```bash
php artisan db:seed
```

To create a Statamic admin user:

```bash
php artisan statamic:make:user
```

## Running Locally

Using the built-in dev command (starts the server, queue worker, log watcher, and Vite):

```bash
composer dev
```

Or run the server on its own:

```bash
php artisan serve
```

The API is available at `http://localhost:8000/api` and the CMS control panel at `http://localhost:8000/cp`.

## Running Tests

```bash
php artisan test
```

Or via Composer:

```bash
composer test
```

## Key Environment Variables

| Variable | Description |
|---|---|
| `APP_KEY` | Laravel encryption key (generated by `key:generate`) |
| `APP_URL` | Base URL of the backend (e.g. `http://localhost` or production domain) |
| `DB_DATABASE` | Path to the SQLite database file (default: `/data/database.sqlite`) |
| `STATAMIC_LICENSE_KEY` | Statamic license key (optional for Core free tier in development) |
| `STRIPE_KEY` | Stripe publishable key |
| `STRIPE_SECRET` | Stripe secret key |
| `STRIPE_WEBHOOK_SECRET` | Stripe webhook signing secret (for verifying webhook payloads) |
| `RESEND_API_KEY` | Resend API key for transactional emails |
| `ROYAL_MAIL_API_KEY` | Royal Mail Click & Drop API key |
| `CONTACT_FORM_RECIPIENT` | Email address that receives contact form submissions |

In local development, email defaults to the `log` driver (emails appear in `storage/logs`). Stripe keys can use Stripe's test mode keys.

## Database

The application uses SQLite with WAL mode enabled. In production, the database lives at `/data/database.sqlite` on a Railway persistent volume. Litestream continuously replicates it to Cloudflare R2 for backup.

Locally, you can use the default `database/database.sqlite` path or point `DB_DATABASE` wherever you prefer.

## API Endpoints

### Products (rate-limited: `api-public`)

- `GET /api/products` - list all products
- `GET /api/products/categories` - list product categories
- `GET /api/products/{slug}` - get a product by slug

### Cart (rate-limited: `api-cart`)

Cart endpoints use a session-based cart via the `ResolveCartSession` middleware.

- `GET /api/cart` - get the current cart
- `POST /api/cart/items` - add an item to the cart
- `PATCH /api/cart/items/{cartItem}` - update item quantity
- `DELETE /api/cart/items/{cartItem}` - remove an item

### Checkout (rate-limited: `api-checkout`)

- `POST /api/checkout` - create a Stripe Checkout session and return the redirect URL

### Contact (rate-limited: `api-contact`)

- `POST /api/contact` - submit the contact form

### Webhooks

- `POST /webhook/stripe` - Stripe payment webhook (verified via `VerifyStripeSignature` middleware)

## CMS Control Panel

The Statamic control panel is at `/cp`. The site owner uses it to:

- Edit page content with the Bard block editor (drag/drop blocks, images, text)
- Add and edit products (name, description, price, sizes, images)
- View incoming orders

The CMS uses Blade templates (not Statamic's Antlers) to reduce lock-in. Content is stored as flat files in the `content/` directory and version-controlled via git. Transactional data (orders, customers, carts) lives in SQLite.
